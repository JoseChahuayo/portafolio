Semana 1

Tecnologías de Desarrollo Web

Definiciones

Las tecnologías de desarrollo web engloban una diversidad de recursos esenciales que comprenden herramientas, lenguajes de programación, frameworks y estándares. Estos elementos son empleados de manera integral en el proceso de creación y diseño de aplicaciones, así como en la elaboración de sitios web que se despliegan en el vasto universo de internet. La utilización coordinada de estas tecnologías permite a los desarrolladores estructurar, diseñar y optimizar tanto la funcionalidad como la apariencia de las plataformas digitales, contribuyendo así a la creación de experiencias interactivas y eficientes para los usuarios en línea.

Procedimientos

Las tecnologías de desarrollo web se desglosan en dos componentes fundamentales: el Lado Frontend y el Lado Backend.
En el Lado Frontend, se encuentran los lenguajes de programación principales como HTML, CSS, y JavaScript. Estos desempeñan un papel crucial en la creación de la interfaz de usuario y la presentación visual de las aplicaciones y sitios web. HTML proporciona la estructura básica del contenido, CSS define el estilo y el diseño, mientras que JavaScript añade interactividad y dinamismo a la experiencia del usuario.
En el Lado Backend, nos adentramos en la infraestructura que respalda el funcionamiento del sitio web. Aquí, nos encontramos con servidores web y una variedad de lenguajes de programación, entre los que destacan PHP, Java, Python, entre otros. Estos lenguajes son responsables de gestionar la lógica empresarial, la interacción con bases de datos, y la respuesta a las solicitudes del usuario.

Resultados

Se adquirió la habilidad de distinguir entre diversas tecnologías web que posibilitan a los desarrolladores la creación de aplicaciones web efectivas y funcionales. Este conocimiento abarca desde la concepción de la estructura y el diseño, hasta la implementación de la funcionalidad y la gestión eficiente de datos. En esencia, se ha obtenido una comprensión integral que abarca todo el espectro del desarrollo web, permitiendo a los profesionales abordar cada aspecto crítico, desde la apariencia visual hasta la experiencia del usuario y el manejo robusto de la información.

Además, HTML posibilita la creación de hipervínculos o enlaces (links) y la implementación de formularios interactivos, permitiendo a los usuarios visitar páginas, proporcionar datos o expresar comentarios.

Asimismo, se reconoce la versatilidad de HTML al posibilitar la creación de interfaces interactivas y atractivas en el vasto panorama de la web.

Semana 7

Uso de hooks

Definiciones

Los Hooks en React son funciones que permiten a los componentes funcionales tener características de los componentes de clase, como el estado y los ciclos de vida, sin necesidad de convertirlos en clases. Los Hooks se utilizan dentro de los componentes funcionales para agregar y manipular el estado y realizar acciones específicas en diferentes momentos del ciclo de vida de un componente. El propósito y utilidad de estos son: 1. Manejo del Estado en Componentes Funcionales, los Hooks, especialmente "useState", permiten a los componentes funcionales tener su propio estado local. 2. Ejecución de Efectos Secundarios, Hooks como "useEffect" permiten realizar acciones secundarias, como llamadas a API o manipulación del DOM, después de que el componente ha sido renderizado. 3. Contexto de React, el Hook useContext permite a los componentes funcionales suscribirse al contexto de React, permitiendo un acceso fácil a datos compartidos en la aplicación.

Procedimientos

El uso de Hooks en React implica la incorporación de funciones especiales dentro de componentes funcionales para gestionar el estado, los efectos y otras funcionalidades. Para trabajar con estos en React se realiza los siguientes pasos: Se importa la función useState de React y se declara el estado utilizando useState y asigna un valor inicial. Para el useEffect se importa la función useEffect de React y se define la función que contiene los efectos secundarios y pasa esta función a useEffect. En el useContext se accede al contexto utilizando useContext y nos proporciona el contexto que se desea utilizar. Para useReducer se define una función reductora y un estado inicial, para luego llamar a useReducer y obtener el estado actual y la función de despacho. Además de los Hooks mencionados, React proporciona varios otros Hooks como useCallback, useMemo, useRef, etc. Cada uno tiene su propio propósito específico.

Resultados

El uso de Hooks en React para el desarrollo de páginas web puede tener varios resultados beneficiosos que mejoran la eficiencia del desarrollo, la organización del código y la experiencia del usuario. En resumen, el uso de Hooks en React ofrece una serie de beneficios que mejoran la eficiencia del desarrollo, la legibilidad del código y la capacidad de crear aplicaciones web más potentes y reactivas. La elección de qué Hooks utilizar dependerá de los requisitos específicos de cada componente y la naturaleza de la aplicación.

Semana 9

Next JS CSR / SSR

Definiciones

Next.js es un framework de desarrollo web React que simplifica y agiliza la creación de aplicaciones web. Proporciona un conjunto de utilidades y convenciones para la creación de rutas, gestión del estado y la posibilidad de renderizar páginas tanto en el lado del servidor como en el lado del cliente. Dentro de su uso y utilidad están: 1. Interactividad Rápida: Permite una carga inicial rápida, ya que solo se envía el esqueleto de la aplicación al cliente, y luego se realiza la renderización en el navegador. 2. Aplicaciones de Una Página (SPA): Adecuado para aplicaciones que requieren una interactividad intensiva en el cliente y funcionan como SPAs, donde las transiciones entre páginas se gestionan sin recargar la página completa.

Procedimientos

Next.js utiliza una serie de procedimientos y conceptos para su ejecución. Aquí una visión de los procedimientos clave que están involucrados en el funcionamiento de una aplicación que utiliza Next.js: Se debe configurar un proyecto Next.js utilizando la herramienta de línea de comandos create-next-app Luego se ejecuta el comando: npm run dev Después, Next.js utiliza un sistema de enrutamiento basado en archivos. Los archivos en la carpeta pages determinan las rutas de la aplicación. Para la creación de páginas dinámicas y estáticas, utiliza las funciones especiales como getStaticProps y getServerSideProps para controlar la obtención de datos. Se pueden gestionar los estilos de los componentes utilizando módulos CSS, Styled JSX, o integrando con bibliotecas populares como SCSS o Tailwind CSS Next.js permite la creación de API Routes para gestionar solicitudes y respuestas en el servidor. Estos archivos se encuentran en la carpeta pages/api. Para desplegar la aplicación, se puede utilizar diferentes servicios, como Vercel, que tiene una integración estrecha con Next.js. También se puede desplegar la aplicación en otros servicios como AWS, Netlify, entre otros.

Resultados

El uso de Next.js puede proporcionar una experiencia de desarrollo más eficiente, mejor rendimiento, y facilitar la implementación de estrategias avanzadas, como la representación del lado del servidor y el SEO mejorado. La configuración inicial de un proyecto Next.js es rápida y sencilla utilizando create-next-app o configurando manualmente. Además, la recarga en tiempo real y la actualización automática durante el desarrollo local facilitan el desarrollo rápido y eficiente. Se pueden crear API Routes en la carpeta pages/api para gestionar solicitudes y respuestas en el servidor. Esto simplifica la creación de puntos de conexión de API en la aplicación.

En el contexto de Next.js, esto implica hacer solicitudes a servicios externos o internos que proporcionan datos o funcionalidades específicas. El propósito y utilidad para el consumo de una API es: 1. Obtener datos externos que permite a la aplicación Next.js obtener datos dinámicamente desde servicios externos, como una API REST, GraphQL, o bases de datos. 2. Actualizaciones dinámicas de contenido que posibilita la actualización dinámica del contenido de la aplicación en función de los datos recibidos de la API. 3. Permite la integración de Next.js con servicios de terceros, como autenticación, pago, redes sociales, etc.

Procedimientos

Para realizar el consumo de APIs en Next.js, se puede seguir algunos procedimientos comunes. A continuación, se presenta un ejemplo básico utilizando las funciones getServerSideProps y fetch para obtener datos de una API en el lado del servidor (SSR). Primero se importa React y Next.js Luego se implementa la función getServerSideProps para realizar la solicitud a la API y obtener los datos. Finalmente, se puede utilizar los datos recibidos de la API en tu componente para realizar diferentes funciones.

Resultados

El consumo de APIs en Next.js ofrece una serie de resultados beneficiosos que contribuyen al desarrollo de páginas web modernas y eficientes. También, permite crear aplicaciones dinámicas, interactivas y escalables al facilitar la integración de datos y servicios externos en el flujo de trabajo de desarrollo. Por último, el consumo de APIs permite un desarrollo ágil, ya que los datos pueden ser modificados y actualizados independientemente del código de la interfaz de usuario.

Definiciones

Permite la autenticación con diversas estrategias, como redes sociales, proveedores de identidad federados, bases de datos locales y más. GetServerSideProps es una función en Next.js que se utiliza para obtener datos en el lado del servidor (SSR) antes de que una página se renderice. NextAuth.js maneja la complejidad de la autenticación, el manejo de sesiones y la gestión de tokens de acceso, lo que simplifica considerablemente el proceso de incorporar la autenticación en tu aplicación Next.js. Para usar NextAuth.js, generalmente configuras un archivo next-auth.config.js con la configuración de autenticación y luego lo integras en tu aplicación Next.js.

Procedimientos

Para usar NextAuth.js y GetServerSideProps los siguientes pasos son: Se implementa la función getServerSideProps para realizar la lógica del servidor y obtener datos. Luego se implementa la función getServerSideProps para realizar la solicitud a la API y obtener los datos. Crea un archivo next-auth.config.js para configurar NextAuth.js con los proveedores de autenticación que desees utilizar. Crea un archivo pages/api/auth/[...nextauth].js para configurar la API Route de NextAuth.js.

Resultados

La combinación de getServerSideProps y NextAuth.js permite un desarrollo seguro y escalable. Además de proporcionar una base sólida para construir aplicaciones web seguras, dinámicas y eficientes. Se puede autenticar a los usuarios y proporcionar contenido dinámico basado en su estado de autenticación y otros datos del lado del servidor. En resumen, el uso de getServerSideProps y NextAuth.js puede aportar resultados significativos al desarrollo de páginas web al mejorar la seguridad, la eficiencia, y la experiencia del usuario al gestionar la autenticación y la obtención de datos en el lado del servidor.

Semana 12

Desarrollo backend

Definiciones

El desarrollo backend, se refiere a la creación y mantenimiento de la parte de una aplicación web o software que opera en el servidor y se encarga de la lógica empresarial, la manipulación de datos y la interacción con la base de datos. En el contexto de aplicaciones web, el backend es responsable de manejar las solicitudes del cliente, procesar la lógica de la aplicación y enviar las respuestas de vuelta al cliente. Algunos aspectos clave del desarrollo backend incluyen: 1. Lógica de Aplicación: Se implementa la lógica empresarial que define cómo funciona la aplicación. Esto puede incluir la manipulación de datos, la autenticación, la autorización y otras operaciones críticas para el funcionamiento de la aplicación. 2. Interacción con la Base de Datos: Esto implica diseñar y mantener esquemas de bases de datos, ejecutar consultas y asegurar la integridad y consistencia de los datos. 3. Manejo de Solicitudes y Respuestas: Gestión de solicitudes HTTP entrantes y generación de respuestas adecuadas. 4. APIs: Desarrollo de APIs que permiten la comunicación entre el frontend y el backend. Las APIs definen cómo interactúan diferentes partes de la aplicación y facilitan la integración con servicios externos.

Procedimientos

El desarrollo backend implica una serie de procedimientos y prácticas para construir la lógica empresarial de una aplicación, gestionar datos y garantizar el funcionamiento eficiente del lado del servidor. A continuación, algunos procedimientos comunes utilizados en el desarrollo backend: 1. Definición de Requisitos y Arquitectura: Se tiene que comprender los requisitos del sistema y diseñar la arquitectura backend. Esto implica definir cómo se estructurará la aplicación, qué tecnologías se utilizarán y cómo interactuarán los diferentes componentes. 2. Selección de Tecnologías: Elegir el conjunto adecuado de tecnologías y herramientas para el desarrollo backend. Esto puede incluir la selección de un lenguaje de programación, un framework, una base de datos, y otras herramientas relacionadas. 3. Desarrollo de la Lógica de Aplicación: Se implementa la lógica empresarial de la aplicación. Esto incluye la manipulación de datos, la gestión de usuarios, la autenticación, la autorización y otras operaciones que definen el comportamiento funcional de la aplicación. 4. Diseño y Mantenimiento de Bases de Datos: Crear y mantener bases de datos que almacenen la información necesaria para la aplicación. 5. Desarrollo de APIs: Crear las APIs que permitan la comunicación entre el frontend y el backend. 6. Despliegue y Mantenimiento: Se despliega la aplicación en entornos de producción y gestionar las actualizaciones y mantenimiento continuo.

Resultados

Se pudo observar que el desarrollo backend es fundamental para el desarrollo de páginas web ya que proporciona la infraestructura necesaria para la funcionalidad, la seguridad y la eficiencia de una aplicación. Un backend bien diseñado contribuye directamente a la calidad y la experiencia del usuario en una aplicación web. Por último, el desarrollo backend desempeña un papel crucial en el desarrollo de páginas web, y los resultados pueden tener un impacto significativo en la funcionalidad, seguridad y rendimiento de una aplicación.

Semana 13

Desarrollar Aplicaciones con Spring Core y Spring Boot

Definiciones

El desarrollo de aplicaciones con Spring Core y Spring Boot se refiere al uso de dos proyectos clave dentro del ecosistema de Spring Framework, una plataforma de desarrollo para la creación de aplicaciones empresariales en Java. Spring Core, también conocido simplemente como "Spring", es el corazón del framework Spring. Proporciona funcionalidades fundamentales que abordan varios aspectos del desarrollo de aplicaciones empresariales en Java. Spring Boot es un proyecto dentro del ecosistema de Spring que simplifica y acelera el desarrollo de aplicaciones basadas en Spring. Algunas de las características clave de Spring Core y Spring Boot son: 1. Spring Boot ofrece configuración automática sensible al contexto, eliminando la necesidad de configuraciones extensas. Detecta y configura automáticamente componentes y servicios comunes basándose en las dependencias presentes en el proyecto. Ofrece herramientas para gestionar dependencias de manera sencilla, utilizando la plataforma Spring Initializr para generar proyectos listos para usar con las dependencias adecuadas. 2. Spring Core IoC Container es responsable de gestionar y mantener los objetos (beans) de una aplicación. Los desarrolladores definen la configuración y las dependencias, y Spring IoC Container se encarga de la inicialización y la administración de esos objetos. Spring Core facilita la Inyección de Dependencias, lo que permite que los objetos obtengan sus dependencias de manera externa. Esto mejora la modularidad y facilita la prueba unitaria.

Procedimientos

El desarrollo de aplicaciones con Spring Core y Spring Boot sigue una serie de procedimientos estándar que involucran la configuración, el diseño de la arquitectura, la implementación de la lógica de la aplicación y la gestión de dependencias. A continuación, se describen algunos de los procedimientos comunes para el desarrollo de aplicaciones con Spring Core y Spring Boot: Spring Core 1. Se crea un proyecto de aplicación Java y se configuran las dependencias de Spring Core en el archivo de construcción 2. Luego se definen los componentes de la aplicación utilizando anotaciones como @Component, @Service, o @Repository. Estos componentes serán gestionados por el contenedor de Spring. 3. Después se utiliza la Inyección de Dependencias (DI) para proporcionar dependencias a través de anotaciones como @Autowired o mediante métodos de configuración. Spring Boot 1. Desarrollar API Rest con Spring

Definiciones

Desarrollar una API REST con Spring implica utilizar el conjunto de herramientas proporcionado por el framework Spring para construir servicios web que sigan el estilo arquitectónico REST REST es un enfoque que utiliza las operaciones estándar HTTP (GET, POST, PUT, DELETE) para realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en recursos. Algunas funcionalidades de usar un API REST con Spring son: 1. Interacción entre Aplicaciones: Las API REST permiten la comunicación y la interoperabilidad entre diferentes sistemas y aplicaciones, ya que utilizan estándares HTTP ampliamente aceptados. 2. Acceso a Datos: Facilita el acceso a datos a través de endpoints específicos, permitiendo la consulta y manipulación de información almacenada en el servidor. 3. Integración con Plataformas Móviles y Frontends: Las API REST son ideales para ser consumidas por aplicaciones móviles y clientes web, proporcionando una interfaz uniforme y fácil de consumir.

Procedimientos

Para realizar el consumo de APIs REST con Spring, se puede seguir algunos procedimientos comunes: Utilizar Spring Initializr para configurar un proyecto Spring Boot con las dependencias necesarias para el desarrollo de API REST. Crear controladores utilizando anotaciones como @RestController para manejar las solicitudes HTTP y definir las operaciones CRUD. Utilizar anotaciones como @RequestMapping, @GetMapping, @PostMapping, @PutMapping, y @DeleteMapping para mapear los endpoints de la API a métodos en los controladores. Configurar la serialización y deserialización de objetos a formato JSON o XML utilizando librerías como Jackson. Documentar la API utilizando herramientas como Swagger o Springdoc para describir los endpoints y facilitar su consumo.

Resultados

Desarrollar una API REST con Spring para el desarrollo de aplicaciones web puede tener varios resultados positivos que contribuyen a la eficiencia, escalabilidad, seguridad y flexibilidad de la aplicación. Una API RESTful permite que la aplicación web interactúe fácilmente con otros sistemas y servicios, independientemente de la tecnología utilizada. Esto facilita la integración con diversas plataformas y componentes. Permite que el desarrollo del frontend sea independiente del backend, siempre y cuando la API siga las especificaciones definidas. Esto favorece la flexibilidad y la agilidad en el desarrollo. En resumen, el desarrollo de una API REST con Spring para aplicaciones web puede tener un impacto positivo en la eficiencia, la colaboración, la escalabilidad y la seguridad del sistema.
